MSc project work log - Self Organizing Feature Map 3

2020-07-02 MSc project Comments

Self Organizing Feature Map 3

This blog demonstrates the comparison of a serial experiment. It will just display the plots. Here we did the experiment to see if the clustering would change greatly when we shuffle the data before feed them into SOFM.

In these experiment, we use the fix hyperparameter of the SOFM and only change the sequence of the input data. The code we use is display below.

OIP level of each neuron

The first pair is the OIP level of each neuron. These plots are training with the default sequence which is sorted by the genetic sequence as it in the OilInPlaceEvolution.csv file. The strategy of estimating neuron’s OIP is using the closest geological model and let the model’s OIP be the neuron’s. (matching strategy).

The second pair is trained by shuffle sequence. The strategy is matching strategy too.

The strategy of estimating neuron’s OIP is using the Gradient Boosting regressor to predict it.

We can see the sequence of training data could effect the neurons, including the value and distance.

The heatmap of neuron’s OIP predicted by Gradient Boosting regressor (predicted strategy) shows the similar circumstance. The first pair is trained by order sequence and the second one is by shuffle data.

Clustering performance

These plot pairs are showing the performance of SOFM clustering. The first 2 plots are the output of the model trained by order sequence. The 2 plots demonstrate the performance that we could say not bad. But there are still some data points are significant clustered in wrong groups.

Sequence input

The first plot shows that there are some relatively dark blue color dots (representing the high level of OIP model) on the low level OIP positions on the bottom of the plot.

The neurons using Gradient Boosting regressor to estimating the OIP can also do wrong clustering. We can see there are fade blue dots (representing the low level of the OIP model) on the top of the plot between approximate 6800-10000 model ID.

Shuffle input

The shuffle data trains better model as the plots below showing. Comparing with the first plot of the sequence input, this one below shows better performance since the weights (marked as red crosses on the plot) are more discrete, leading to more distance between the clusters. The dots’ color are basically corresponding with the true OIP level.

The dots clustered by the neurons’ OIP from gradient boosting regressor are much better. Comparing with the neuron’s OIP getting from matching strategy, these dots with a similar colour are showing horizontal distribution, which is what we expect.

Conclusion

From the simple analysis, we can know shuffling the data can improve the performance of the SOFM model.
